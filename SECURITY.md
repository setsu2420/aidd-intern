# 安全策略

AIDD-Intern 团队非常重视安全问题。感谢你负责任地报告安全漏洞。

## 支持的版本

| 版本    | 支持状态           |
| ------- | ------------------ |
| 最新版  | ✅ 接受安全报告    |
| 旧版本  | ❌ 不再维护        |

我们仅对**最新发布版本**提供安全修复。建议用户始终使用最新版本。

## 漏洞报告流程

如果你发现了安全漏洞，请按照以下步骤操作：

1. **通过邮件报告**：发送漏洞详情至 **setsu2420@gmail.com**。
2. **报告内容应包含**：
   - 漏洞的详细描述
   - 复现步骤（尽可能详细）
   - 潜在影响范围
   - 你建议的修复方案（如有）
3. **响应时间**：
   - 我们将在 **48 小时内** 确认收到报告。
   - 我们将在 **7 个工作日内** 提供初步评估和处理计划。
   - 修复完成后，我们会在发布说明中致谢报告者（除非你要求匿名）。

## 注意事项

- ⚠️ **请勿在公开 Issue 或 Discussion 中披露安全漏洞**。公开披露可能会使用户面临风险。
- 请勿利用漏洞访问或修改他人数据。
- 在漏洞修复发布之前，请勿公开漏洞详情。
- 我们承诺不会对善意的安全研究者采取法律行动。

感谢你帮助我们保护 AIDD-Intern 及其用户的安全！
